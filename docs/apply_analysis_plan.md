# План внедрения рекомендаций

Документ сохранён как follow-up к старому анализу. Ниже зафиксирован короткий outcome по каждому пункту, чтобы не использовать его как «живой» backlog без сверки с текущими source-of-truth документами (`README.md`, `docs/index.md`, `docs/technical_manual.md`, runbook-документами Railway).

## Бекенд

1. Подготовить постепенную миграцию кода на TypeScript: настроить сборку и добавить типы для сервисов. — **outcome: выполнено**.
2. Проверить индексы MongoDB на полях фильтрации задач и пользователей. — **outcome: частично выполнено**.
3. Настроить мониторинг размера файла `logs/api.log` и параметра `LOG_BUFFER_SIZE`, при необходимости автоматически архивировать записи старше 30 дней. — **outcome: частично выполнено**.
4. Добавить алерты Prometheus на рост метрик ошибок и CSRF. — **outcome: частично выполнено**.
5. Расширить набор автотестов, покрывая критичные маршруты и проверки ролей. — **outcome: частично выполнено**.
6. Включить `noImplicitAny`, убрать директивы `@ts-nocheck` и описать типы контроллеров и сервисов. — **outcome: частично выполнено**.
7. Заменить все `any` на явные типы, вернуть правило ESLint `no-explicit-any`. — **outcome: частично выполнено**.
8. Удалить модуль `services/tasks.ts`, перенести его функции в `TasksService`. — **outcome: частично выполнено**.
9. Перейти от service locator к инъекциям через конструкторы `tsyringe`. — **outcome: частично выполнено**.
10. Разделить `db/queries.ts` на специализированные репозитории по доменам. — **outcome: частично выполнено**.
11. Типизировать логгер `wgLogEngine` и описать интерфейсы выходов. — **outcome: частично выполнено**.
